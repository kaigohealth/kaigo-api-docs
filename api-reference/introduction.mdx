## Overview

The Kaigo Platform Healthcare API provides secure, HIPAA-compliant endpoints for voice agent interactions in healthcare settings. This reference documents all available endpoints, request/response formats, and error codes.

<Note>
  All endpoints are auto-generated from our [OpenAPI specification](/kaigo-openapi-spec.yaml). You can test endpoints directly in the documentation using the API playground.
</Note>

## Authentication

All API requests require authentication. The Kaigo API uses a multi-layered authentication approach:

<ParamField
  header="Authorization"
  type="string"
  placeholder="Bearer your_api_token"
  required
>
  Service-level authentication using bearer token. Required for all API requests.
</ParamField>

<ParamField
  header="X-Service-ID"
  type="string"
  placeholder="your_service_id"
>
  Service identifier for certain endpoints. Contact support for your Service ID.
</ParamField>

<ParamField
  header="X-Patient-Session"
  type="string"
  placeholder="session_token_from_verification"
>
  Patient session token obtained from verification. Required for accessing PHI endpoints.
</ParamField>

## Response Format

All successful responses follow this structure:

```json
{
  "data": {
    // Response data
  },
  "meta": {
    "request_id": "req_123abc",
    "timestamp": "2024-01-15T10:30:00Z"
  }
}
```

## Error Responses

Error responses include detailed information for debugging:

```json
{
  "error": "error_code",
  "message": "Human-readable error message",
  "detail": "Additional context about the error",
  "timestamp": "2024-01-15T10:30:00Z"
}
```

### Common Error Codes

| Status | Code | Description |
|--------|------|-------------|
| `401` | `unauthorized` | Invalid or missing authentication |
| `403` | `session_mismatch` | Session doesn't authorize this resource |
| `404` | `not_found` | Resource not found |
| `410` | `session_expired` | Patient session has expired |
| `429` | `rate_limited` | Too many requests |

## Rate Limiting

The API implements intelligent rate limiting to ensure service stability:

| Endpoint Type | Limit | Window |
|--------------|-------|---------|
| Verification | 5 attempts | 15 minutes |
| PHI Access | 100 requests | 1 minute |
| General API | 1000 requests | 1 minute |

Rate limit headers are included in all responses:

```http
X-RateLimit-Limit: 100
X-RateLimit-Remaining: 95
X-RateLimit-Reset: 1705318200
```

## Idempotency

For critical operations, use idempotency keys to prevent duplicate actions:

```bash
curl -X POST https://api.kaigo.com/api/v1/admin/calls/results \
  -H "X-Idempotency-Key: unique_key_123"
```

## Pagination

List endpoints support pagination using cursor-based pagination:

```bash
GET /api/v1/admin/calls/scheduled?limit=50&cursor=eyJpZCI6MTIzfQ
```
